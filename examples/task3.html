A non-empty zero-indexed array A consisting of N integers is given.

We wish to consider an integer Y, which lies between the minimal and maximal values in array A. More precisely, min(A[0], A[1], ..., A[N − 1]) ≤ Y ≤ max(A[0], A[1], ..., A[N − 1]).

For each value of Y, the size of its gap is the distance from Y to the closest value in array A. More precisely, the size of the gap equals min(|A[0] − Y|, |A[1] − Y|, ..., |A[N − 1] − Y|). Our goal is to find the largest possible gap.

For example, consider array A such that:

  A[0] = 10
  A[1] = 0
  A[2] = 8
  A[3] = 2
  A[4] = -1
  A[5] = 12
  A[6] = 11
  A[7] = 3

For both values Y = 5 and Y = 6 the gap size equals 2, since min(|3 − 5|, |8 − 5|) = min(|3 − 6|, |8 − 6|) = 2, and this is the largest possible gap size.

Assume that:

  • N is an integer within the range [2..100,000];
  • each element of array A is an integer within the range [−1,000,000,000..1,000,000,000].

Complexity:

  • expected worst-case time complexity is O(N*log(N));
  • expected worst-case space complexity is O(N), beyond input storage (not counting the storage required for input arguments).

Elements of input arrays can be modified.
